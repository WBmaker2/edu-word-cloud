# 교사용 워드 클라우드 사이트 설계

## 1. 목표

교사가 학생 답변, 수업 소감, 독서 감상문, 모둠 의견 등의 텍스트를 붙여넣고 수업 중 빠르게 워드 클라우드로 시각화할 수 있는 무광고 웹사이트를 만든다.

성공 기준은 다음과 같다.

- 회원가입과 광고 없이 바로 사용할 수 있다.
- 입력 텍스트는 브라우저 안에서만 처리하며 서버로 전송하거나 저장하지 않는다.
- 처음 방문한 교사도 별도 설명 없이 3분 안에 결과를 만들고 PNG로 저장할 수 있다.
- 데스크톱, 태블릿, 모바일에서 핵심 기능을 사용할 수 있다.
- 원, 말풍선, 하트, 별, 책의 다섯 가지 마스크만 제공해 선택 부담을 줄인다.

## 2. 참고 사이트에서 가져올 원칙

`https://wordcloud.kr/v1`에서 다음 제품 원칙을 참고한다.

- 텍스트 입력 후 즉시 워드 클라우드를 만들 수 있는 단순한 흐름
- 색상, 글꼴, 마스크, 크기, 단어 수 등 결과를 조절하는 기능
- 특정 키워드를 더 크게 표시하는 기능
- 완성 결과를 이미지로 내려받는 기능

참고 사이트의 광고, 공개 갤러리, 많은 선택지, 오래된 표현과 복잡한 화면 구조는 가져오지 않는다. 디자인과 구현은 독립적으로 제작한다.

## 3. 사용자와 대표 사용 흐름

주 사용자는 초중등 교사다. 대표 흐름은 다음과 같다.

1. 교사가 사이트에 접속한다.
2. 개인정보가 서버에 저장되지 않는다는 안내를 확인한다.
3. 학생 답변이나 수업 자료를 입력창에 붙여넣는다.
4. 불필요한 단어를 제외하거나 핵심어를 최대 3개 지정한다.
5. `워드 클라우드 만들기`를 눌러 큰 미리보기에서 결과를 확인한다.
6. 원, 말풍선, 하트, 별, 책 중 마스크 하나를 선택한다.
7. 색상, 글꼴, 표시할 단어 수를 바꾸면 미리보기가 즉시 갱신된다.
8. 결과를 PNG 이미지로 저장해 수업 자료에 사용한다.

## 4. 화면 설계

선택된 디자인 방향은 `큰 미리보기 중심`이다.

### 4.1 상단 영역

- 서비스 이름: `클라우드 수업실`
- 짧은 설명: `학생들의 생각을 한눈에 모아보세요.`
- 상태 표지: `광고 없음`, `서버 저장 없음`
- 보조 버튼: `도움말`, `업데이트 내역`

### 4.2 큰 미리보기 영역

- 첫 화면에서 가장 넓은 면적을 차지한다.
- 입력 전에는 예시 워드 클라우드와 시작 안내를 표시한다.
- 입력 후에는 선택한 마스크 안에 배치된 단어를 표시한다.
- 미리보기 상단에는 결과 제목과 `PNG 저장` 버튼을 둔다.
- 모바일에서도 가로 스크롤 없이 전체 결과를 확인할 수 있도록 크기를 맞춘다.

### 4.3 빠른 설정 도구

미리보기 위 또는 바로 아래에 다음 항목을 짧은 탭 또는 버튼 묶음으로 배치한다.

- 마스크: 원, 말풍선, 하트, 별, 책
- 색상: 교실 초록, 맑은 파랑, 따뜻한 주황, 다채로운 무지개
- 글꼴: 깔끔한 고딕, 힘 있는 고딕, 부드러운 명조
- 단어 수: 20개, 40개, 60개, 80개, 100개

단어 수의 기본값은 `40개`로 설정한다.

선택한 항목은 색상과 테두리뿐 아니라 체크 표시와 설명 텍스트로도 구분한다.

### 4.4 텍스트 입력과 단어 정리

- 큰 텍스트 입력창과 글자 수 안내를 제공한다.
- `독서 감상 예시`, `수업 소감 예시`, `환경 수업 예시`를 한 번에 불러올 수 있다.
- `워드 클라우드 만들기` 또는 `다시 만들기` 버튼을 제공한다.
- 한국어 기본 불용어를 자동으로 제외한다.
- 사용자가 제외할 단어를 쉼표나 줄바꿈으로 추가할 수 있다.
- 핵심어를 최대 3개 지정하고 입력 순서에 따라 강조 비율을 다르게 적용한다.
- 상위 단어와 빈도를 표로 보여준다.

### 4.5 도움말과 업데이트 내역

- 도움말은 첫 사용자가 전체 흐름을 이해할 수 있는 짧은 설명 패널로 제공한다.
- 업데이트 내역은 화면을 가리지 않는 작은 버튼으로 열 수 있다.
- 최초 기록: `2026-07-19 — 교사용 워드 클라우드 사이트 첫 제작`

## 5. 기능 설계

### 5.1 텍스트 분석

- 한글, 영문, 숫자로 구성된 단어를 추출한다.
- 문장 부호와 단독 숫자, 한 글자 일반어를 기본적으로 제외한다.
- 대소문자를 통합하고 같은 단어의 출현 횟수를 계산한다.
- 기본 불용어와 사용자 제외어를 제거한다.
- 핵심어는 실제 빈도와 관계없이 최소 강조 가중치를 부여한다.
- 빈도와 핵심어 가중치를 기준으로 내림차순 정렬한다.

### 5.2 마스크와 배치

- 모든 마스크는 수학 함수로 정의된 내부 영역을 사용하며 외부 이미지를 업로드하지 않는다.
- 마스크 종류는 원, 말풍선, 하트, 별, 책의 다섯 가지다.
- 큰 단어부터 중심에 가깝게 배치하고 충돌하는 위치는 나선형으로 탐색한다.
- 가로 단어를 기본으로 하되 일부 보조 단어만 세로로 회전한다.
- 배치할 공간이 부족하면 작은 단어부터 생략하고 사용자에게 실제 표시 개수를 알린다.
- 같은 입력과 설정에서는 같은 결과가 나오도록 결정적 난수 시드를 사용한다.

### 5.3 이미지 저장

- 현재 미리보기를 배경색과 함께 PNG로 저장한다.
- 기본 저장 크기는 1200×800픽셀이다.
- 파일명은 `클라우드-수업실-YYYY-MM-DD.png` 형식을 사용한다.
- 빈 결과에서는 저장 버튼을 비활성화하고 이유를 안내한다.

## 6. 기술 구조

사이트는 React 기반 단일 페이지로 구성하며 Cloudflare Worker 호환 Sites 빌드 구조를 유지한다. 데이터베이스, 인증, 외부 API는 사용하지 않는다.

책임을 다음 단위로 분리한다.

- 페이지 조립과 전체 상태 관리
- 텍스트 분석과 단어 빈도 계산
- 마스크 내부 판정
- 단어 위치 계산과 충돌 검사
- Canvas 렌더링과 PNG 저장
- 설정 도구, 입력 패널, 빈도표, 도움말, 업데이트 내역 UI

각 코드 파일은 500줄 미만을 유지한다. 브라우저 저장소에는 마지막으로 선택한 마스크, 색상, 글꼴, 단어 수만 저장하며 사용자가 입력한 원문, 제외어, 핵심어, 분석 결과는 저장하지 않는다.

## 7. 데이터 흐름

1. 텍스트와 사용자 제외어, 핵심어를 입력받는다.
2. 분석 함수가 정규화된 단어와 빈도 목록을 만든다.
3. 선택한 단어 수와 핵심어 가중치를 반영해 표시 목록을 만든다.
4. 배치 함수가 선택한 마스크 안에서 충돌하지 않는 좌표를 계산한다.
5. 렌더러가 Canvas에 배경, 단어, 마스크 장식을 그린다.
6. 같은 Canvas를 미리보기와 PNG 저장에 사용한다.

원문 텍스트와 분석 결과는 현재 브라우저 메모리에만 존재한다. 새로고침하면 원문은 사라진다.

## 8. 오류와 안내

- 빈 입력: `먼저 학생 답변이나 수업 내용을 붙여넣어 주세요.`
- 의미 있는 단어 부족: `두 글자 이상의 단어를 조금 더 입력해 주세요.`
- 핵심어 3개 초과: 네 번째 입력을 막고 제한 이유를 표시한다.
- 너무 긴 입력: 분석 가능한 길이를 50,000자로 제한하고 초과 부분이 처리되지 않았음을 알린다.
- 마스크 공간 부족: 실제 표시된 단어 수를 결과 아래에 표시한다.
- PNG 저장 실패: 다시 시도할 수 있는 안내를 제공한다.

오류 안내는 색상만으로 구분하지 않고 제목과 해결 방법을 함께 표시한다.

## 9. 접근성과 반응형

- 모든 설정 버튼에 의미 있는 이름과 선택 상태를 제공한다.
- 키보드만으로 입력, 설정 변경, 생성, 저장, 패널 닫기가 가능해야 한다.
- 본문과 버튼은 WCAG AA 수준의 명도 대비를 목표로 한다.
- 포커스 표시를 제거하지 않는다.
- 모바일에서는 `미리보기 → 설정 → 입력 → 빈도표` 순으로 세로 배치한다.
- 움직임 최소화 설정을 존중하고 필수적이지 않은 애니메이션은 사용하지 않는다.

## 10. 테스트 전략

### 단위 테스트

- 한국어와 영문 단어 추출 및 빈도 계산
- 기본 불용어와 사용자 제외어 제거
- 핵심어 가중치와 정렬
- 다섯 마스크의 내부·외부 좌표 판정
- 같은 입력에서 같은 배치 결과가 나오는지 확인
- 잘못된 입력과 길이 제한 처리

### 통합 검증

- 초기 화면에 서비스 설명과 개인정보 안내가 표시되는지 확인
- 예시 텍스트로 워드 클라우드가 생성되는지 확인
- 다섯 마스크를 바꿀 때 결과가 다시 계산되는지 확인
- 빈 입력에서 생성과 저장이 차단되는지 확인
- 업데이트 내역에 최초 제작 기록이 표시되는지 확인
- 배포용 빌드가 성공하는지 확인

## 11. 제외 범위

첫 버전에는 다음 기능을 넣지 않는다.

- 학생 실시간 응답 링크
- 로그인과 계정 저장
- 공개 갤러리와 결과 공유 피드
- 사용자 이미지 마스크 업로드
- PDF, SVG, 프레젠테이션 파일 내보내기
- 형태소 분석 외부 API
- 광고와 제휴 링크

이 범위는 교사가 개인정보 걱정 없이 즉시 사용하는 핵심 경험에 집중하기 위한 결정이다.
