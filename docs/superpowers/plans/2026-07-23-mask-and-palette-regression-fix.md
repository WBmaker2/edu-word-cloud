# 마스크·색상 선택 회귀 수정 계획

## 확인된 원인

1. 단어 배치는 `masks.mjs`의 도형 판정을 사용하지만, 캔버스 테두리는 원·말풍선 외에 항상 둥근 사각형을 그린다. 따라서 별·하트·책 선택이 시각적으로 반영되지 않는다.
2. 배치 계산용 캔버스 비율(1200×500)과 CSS 표시 비율이 달라, 원 테두리와 실제 단어 배치 범위가 어긋난다.
3. 선택하지 않은 색상 견본은 퍼센트 높이를 계산할 기준 높이가 없어 0px로 렌더링된다. 체크 표시가 있는 선택 견본만 내용 높이를 가져 색이 보인다.

## 구현 범위

1. 마스크별 공통 경계와 경로를 `masks.mjs`에서 제공하고, 배치 검사와 캔버스 윤곽선이 같은 규칙·크기를 쓰도록 한다.
2. 캔버스를 실제 비율로 표시해 렌더링한 도형과 단어 배치 좌표가 일치하게 한다.
3. 모든 색상 선택지에 고정 크기의 다색 견본을 표시하고, 선택 여부는 체크 표시와 테두리로만 구분한다.
4. 마스크 기하와 색상 견본의 회귀 테스트를 보강하고, 업데이트 내역에 2026-07-23 개선 기록을 추가한다.

## 검증

- 단위·렌더링 테스트와 정적 빌드를 실행한다.
- 배포 화면에서 별·원 선택, 다색 견본의 상시 노출, 데스크톱·모바일 표시를 확인한다.
