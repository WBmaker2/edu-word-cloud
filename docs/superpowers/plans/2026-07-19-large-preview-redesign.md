# 큰 미리보기 중심 UI 개선 계획

## 기준 시안

- `output/imagegen/teacher-word-cloud-redesign-concept.png`
- 초기 승인 방향을 유지한다: 첫 화면의 가장 큰 면적은 워드 클라우드 미리보기이며, 설정은 바로 아래의 한 줄 도구막대에 둔다.

## 디자인 시스템

- 배경: true white `#ffffff`, 보조 면: 옅은 하늘색 `#f4f8ff`
- 잉크: 짙은 남색 `#11284b`, 주요 동작: 하늘색 `#4d91f7`, 생성 버튼: 산호색 `#ef6f61`
- 테두리: `#d6e1f0`, 반경: 16px, 그림자: 가장 약한 1단계만 사용한다.
- 제목은 여유 있는 큰 고딕, 도구·입력 요소는 14~15px의 명확한 고딕을 사용한다.

## 구현 항목

1. 헤더를 작은 브랜드 줄로 정리하고 개인정보 안내와 보조 동작을 과장 없이 배치한다.
2. 미리보기 패널을 화면의 중심으로 확장하고, 저장 동작을 캔버스 설명과 같은 줄에 배치한다.
3. 네 종류의 빠른 설정을 카드 그리드 대신 한 개의 가로 도구막대로 재구성한다.
4. 입력 작업 공간과 빈도표를 데스크톱에서 두 칸으로 배치하고, 모바일에서는 미리보기·설정·입력·표 순으로 자연스럽게 쌓는다.
5. 기존의 분석, 마스크, 저장, 접근성, 로컬 저장 범위는 바꾸지 않는다.
6. 업데이트 내역에 2026-07-19 UI 개선 기록을 추가한다.

## 검증

- 기존 단위·렌더링 테스트와 정적 빌드를 실행한다.
- 브라우저에서 예시 입력, 설정 선택, 생성, PNG 저장 버튼 상태를 확인한다.
- 데스크톱과 모바일 화면을 캡처해 시안과 나란히 `view_image`로 비교한다.
